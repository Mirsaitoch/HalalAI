# Трекер экспериментов RAG — качество ответов

## Гипотезы

1. **Гипотеза 1 — влияние RAG**: Использование retrieval-механизма повышает качество ответов за счёт опоры на релевантные аяты
2. **Гипотеза 2 — влияние fine-tuning embedding**: Дообучение embedding модели на кораническом корпусе повышает качество поиска
3. **Гипотеза 3 — влияние представления данных**: Использование чанков (несколько аятов) повышает полноту ответов

## Методология оценки

### Критерии (по шкале 0–5)

- **Корректность (A)** — Насколько ответ соответствует содержанию Корана
  - 0 = полное несоответствие, 5 = идеальное соответствие

- **Обоснованность (G)** — Насколько ответ опирается на релевантные источники
  - 0 = полное несоответствие, 5 = идеальное соответствие

- **Полнота (C)** — Насколько полно раскрыт вопрос
  - 0 = полное несоответствие, 5 = идеальное соответствие

- **Галлюцинации (H)** — Наличие недостоверной информации
  - 0 = их полное отсутствие, 5 = большое количество

### Формула итогового score

```
Score = 0.4×A + 0.3×G + 0.2×C − 0.3×H
```

Вес:
- Корректность: 40% (самый важный критерий)
- Обоснованность: 30% (опора на Коран)
- Полнота: 20% (охват темы)
- Галлюцинации: −30% (штраф за выдуманное)

## Training Data

### Состав
- **Исходные пары**: 8 (после удаления 4 проблемных)
- **Расширено из data.txt**: 107 пар
- **Объединено**: 115 уникальных пар
- **После добавления вариаций с несколькими аятами**: 189 пар (1.6 пары/вопрос)

### Файл
- `tests/fixtures/quranic_pairs_expanded_multi.json` (189 пар)
- Каждая пара: query, relevant (из Корана), irrelevant, source (сура:аят)

### Fine-tuning параметры
- **Batch size**: 4 (оптимизировано для MPS памяти)
- **Epochs**: 20
- **Learning rate**: 2e-5 (консервативно)
- **Warmup steps**: 50
- **Loss**: CosineSimilarityLoss

## Инструкции по использованию

1. Запустить `python scripts/run_experiments.py`
2. Для каждой конфигурации будут сохранены retrieved verses в `results/C{N}/question_{M}.txt`
3. Оценить ответы по критериям вручную
4. Заполнить таблицу выше
5. Вычислить итоговый score по формуле
6. Записать выводы в "Примечания по результатам"
